# Handoff: Radi coffee — мини-апп лояльности

## Overview
Программа лояльности для кофейни: каждые 6 чашек — одна бесплатно (бесконечно).
Заменяет бумажные визитки. Бариста находит гостя по имени/номеру (или скану QR) и
отмечает кофе в один тап. Приложение задумано как **мини-апп внутри Telegram / MAX**
(никакого нативного приложения пока не планируется).

Три роли в одном приложении, экран определяется ролью аккаунта:
- **Клиент** — своя карта лояльности + QR.
- **Бариста** — поиск гостей и отметка кофе.
- **Владелец** — статистика, управление баристами, и полный функционал баристы.

## About the Design Files
Файл `Radi coffee - приложение.dc.html` — это **дизайн-референс, собранный в HTML**
(интерактивный прототип, показывающий внешний вид и поведение). Это **не продакшн-код
для копирования**. Задача — **воссоздать эти экраны в реальной среде**: фронтенд мини-аппа
(Telegram Web App / MAX mini app, обычно React) + бэкенд с общей базой. Если среды ещё нет —
выбрать подходящий стек и реализовать там.

Прототип хранит всё в `localStorage` и имеет демо-наполнение (20 гостей) и демо-переключатель
ролей внизу экрана — **в проде этого нет**: данные живут на сервере, роль выдаёт владелец.

## Fidelity
**High-fidelity.** Финальные цвета, типографика, отступы и взаимодействия. UI следует
воссоздать пиксель-в-пиксель средствами целевого кодбейса. Дизайн рассчитан на мобильный
экран (макет рисовался в рамке 340×720 CSS-px, верстать резиново под ширину телефона).

## Роли и доступ
- Аккаунт создаётся при первом входе (онбординг). Поле `role`: `client` | `barista`.
- **Владелец** — отдельный флаг/роль (в прототипе — третий режим). У владельца есть кнопки
  «Сделать баристой» / «Уволить» на каждом аккаунте.
- Назначение баристой открывает функционал баристы; увольнение — возвращает обычный режим клиента.
- Бариста в кофейне один (но модель поддерживает нескольких).

## Данные (модель аккаунта)
```
Account {
  id: string|number
  name: string         // имя для отображения; ТОЛЬКО кириллица, пробел, дефис
  phone: string        // 10 цифр нац. номера (без +7); хранить нормализованным
  role: 'client' | 'barista'
  count: number        // прогресс текущего цикла, 0..5
  total: number        // всего чашек за всё время (платные + бесплатные)
  bonus: number        // накопленных бесплатных «в запасе», 0..5
  freeGiven: number    // сколько бесплатных уже выдано
}
```
Профиль владельца — отдельная роль/флаг поверх этой модели.

## Онбординг (вход)
1. Экран 1: логотип + «Поделиться номером». Номер берётся из Telegram/MAX initData —
   **вручную не вводится**. (В прототипе зашит демо-номер.)
2. Экран 2: показывается подтверждённый номер (зелёная точка) + поле «Ваше имя».
   - Имя: фильтр ввода — только `[А-Яа-яЁё пробел дефис]`. Кнопка активна при непустом имени.
3. «Начать копить» → создаётся/привязывается аккаунт, дальше экран по роли.
Сессия запоминается (в проде — по Telegram user id, не по localStorage).

## Экраны

### 1. Клиент · Моя карта
- Шапка: логотип-плашка «R» + имя + «ваша карта лояльности».
- Карточка лояльности (тёмная, рамка `rgba(194,160,121,0.42)`, radius 22):
  заголовок «6-й кофе в подарок ♥», сетка 3×2 «штампов» (5 кружков + 6-й = сердце-подарок),
  адрес «ул. Болотниковская, 36к5».
- Статус: «До подарка осталось: N» или «Заберите 6-й кофе бесплатно ♥».
- Строка: «N выпито · M бесплатных».
- **Снизу QR** (см. раздел QR) с подписью «Покажите QR баристе».

### 2. Бариста
- **Список гостей**: шапка «Гости / РЕЖИМ БАРИСТЫ» + кнопка скана QR (иконка-рамка).
  Поиск по имени или номеру. Строка гостя: аватар-инициал, имя, номер, прогресс-бар (6 сегментов),
  «N/6», бейдж «ПОДАРОК» если count≥5. Сортировка: сначала «горячие» (готовые к подарку), затем по имени.
  Если не найдено — «Гость не найден. Попросите его открыть бота Radi coffee.»
- **Карточка гостя**: аватар, имя, номер, штампы 5+1, статус цикла,
  строка «N чашек всего · M бесплатных · K в запасе».
  Кнопки (снизу):
  - если count<5 → **«+1 кофе»**.
  - если count≥5 → **«Пропустить выдачу»** (вторичная, выше) + **«Выдать 6-й бесплатно ♥»** (основная).
  - если bonus>0 → **«Выдать бесплатный из запаса · N»**.
  Все действия — через **диалог подтверждения**; после действия — баннер **«… · Отменить»** (откат).

### 3. Владелец (нижняя навигация: Статистика · Баристы · Гости)
- **Статистика**: большая плитка «чашек отмечено сегодня», сетка 2×2
  (гостей в базе · подарков выдано · готовы к подарку · барист),
  блок **«Динамика возвращения гостей»** — топ-20 по `total` (убыв.), с мини-баром;
  **строки кликабельны** → открывают карточку гостя. Список — снимок на момент запуска/обновления.
- **Баристы**: по умолчанию список текущих баристов с кнопкой «Уволить».
  Поиск находит любого пользователя из базы для назначения («Сделать баристой»).
  Всех пользователей подряд не показывать — только при поиске.
- **Гости**: полная копия функционала баристы (на случай, если владелец сам встал за стойку).

## Логика начисления
- **+1 кофе**: `count++`, `total++`.
- **Выдать 6-й бесплатно** (при count≥5): `count=0`, `total++`, `freeGiven++`.
- **Пропустить выдачу** (при count≥5; гость оплатил, бесплатный откладываем):
  `count=0`, `total++`, `bonus = min(5, bonus+1)`. Накопление **максимум 5**.
- **Выдать бесплатный из запаса** (при bonus>0): `bonus--`, `total++`, `freeGiven++`.
- **Отмена**: восстановить снимок `{count,total,bonus,freeGiven}` предыдущего состояния.
- Каждое действие отмечается в «сегодня»; отмена — уменьшает.

## QR-код
- В QR зашиваются данные аккаунта (минимум — id; в прототипе строка `name|phone`),
  чтобы бариста при скане **сразу нашёл гостя**.
- В проде — генерировать **настоящий сканируемый QR** (любая библиотека: `qrcode`),
  крупные модули, тёмное на кремовом. В прототипе QR декоративный, с фирменной деталью:
  по центру и в поле — иконки кофейных зёрен (брендинг). Это опционально-косметическое.
- Скан у баристы и у владельца → находит гостя и открывает его карточку.

## Подтверждения и отмена
- Назначение/увольнение баристы — тоже через подтверждение («Уволить» — как опасное действие, красноватое).
- Любая отметка кофе — подтверждение + возможность отката последнего действия.

## State (что нужно)
`onboarded`, `me` (текущий аккаунт), `role`, `accounts` (общая база — с сервера),
`query` (поиск гостей), `mQuery` (поиск для назначения баристой), `ownerTab`,
`selectedId` (открытая карточка), `bScreen` (list|card), `pending` (диалог),
`undo` (снимок для отката), `todayCount`, `loyalty` (снимок рейтинга при запуске),
`scanning` (оверлей скана).

## API (что нужно от бэкенда)
Ориентировочно:
- `POST /auth` — по Telegram/MAX initData → аккаунт (создать при первом входе), вернуть профиль+роль.
- `GET /accounts?query=` — поиск гостей (имя/номер). Для владельца — поиск всех; для назначения.
- `GET /baristas` — список баристов.
- `POST /accounts/:id/cup` — действие: `{type: 'cup'|'redeem'|'skip'|'bonus'}` (атомарно меняет счётчики).
- `POST /accounts/:id/undo` — откат последнего действия (или generic restore).
- `POST /accounts/:id/role` — `{role}` назначить/снять баристу (только владелец).
- `GET /stats/today` и `GET /stats/loyalty` — статистика и рейтинг (снимок).
QR несёт `id` аккаунта → бариста сканирует → `GET /accounts/:id`.

### Typography
- Семейство: **Manrope** (Google Fonts), веса 300–800.
- Числа в счётчиках/номерах: `font-variant-numeric: tabular-nums`; номера телефонов — моноширинный.
- Размеры: заголовки экранов 14px/600; имя в карточке 24px/700; крупная статистика 46px/800;
  кнопки 16px/700; подписи 11–13px.

### Прочее
- Радиусы: рамка телефона 36–46, карточки 16–22, кнопки 15–16, плашки/иконки 8–14.
- Отступы экрана: ~22px по бокам.
- Полосы прокрутки скрыты (`scrollbar-width:none`).
- Номер телефона: ввод нормализуется — отбрасываются `+`, `+7`, `7`, `8`, `+8` в начале;
  хранится 10 цифр; отображается как `+7 916 285-48-27`.
- Hit-target кнопок ≥ 44px.
